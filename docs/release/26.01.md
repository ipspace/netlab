## Breaking changes

Release 26.01 includes numerous potentially-breaking changes. While most of them should not affect casual _netlab_ users, some might break additional tooling one might have built around _netlab_. We apologize for the inconvenience :(

### Configuration Template Changes

* _netlab_ no longer uses Ansible to generate device configuration files. The device configuration templates thus cannot use most Ansible-specific Jinja2 filters ([more details](dev-templates))

```{tip}
You can use the undocumented `--generate ansible` parameter in the **‌netlab initial** command to generate configuration files with Ansible. To use Ansible to generate configuration files while starting a lab, use **‌netlab up --no-config** followed by **‌netlab initial --generate ansible**.
```

* The **ipaddr** filter behavior has been modified: when called with an integer argument (e.g., `ipaddr(0)`), it now returns a CIDR prefix (e.g., `10.0.0.1/24`) instead of just an IP address (e.g., `10.0.0.1`). This makes the behavior consistent with Ansible's ipaddr filter.

* Device configuration templates now have access to additional template variables that were previously only available in Ansible playbooks: **inventory_hostname**, **netlab_device_type**, **node_provider**, and **netlab_interfaces**.

* Template variable names: The **hosts** variable (mapping node names to IP addresses) is being renamed to **host_addrs** for consistency with Ansible. Both variable names are currently available to maintain backward compatibility, but **hosts** may be deprecated in a future release.

### Configuration Deployment Changes

* **netlab initial** now automatically refreshes node configuration files if the underlying templates have changed. Use the `--no-refresh` flag to disable this behavior (this flag is automatically used by **netlab up** to avoid unnecessary file regeneration).

* Configuration deployment to containerlab nodes now aborts on the first error instead of continuing to other nodes. This provides faster feedback when configuration issues occur.

* **netlab initial** and **netlab config** commands now validate that the specified nodeset is not empty and will exit with an error message if no nodes match the selection criteria.

### netlab config Command Changes

* The **netlab config** command has been completely rewritten to support Ansible 12+ while maintaining compatibility with earlier versions. Configuration templates are now rendered by _netlab_'s internal templating engine before being deployed via Ansible.

* The `-e` (extra variables) parameter in **netlab config** now only supports the `name=value` format. Previously, Ansible's `-e` parameter also supported JSON format (`-e '{"key":"value"}'`) and file references (`-e @file.yml`). Only the simple `name=value` format is now supported (e.g., `-e var1=value1 -e var2=value2`).

* When using **netlab config --reload**, extra variables (`-e` parameter) are no longer allowed, as the command reloads previously saved configurations.

### File and Directory Changes

* **clab_files** directory has been renamed to **node_files** and now contains all node configuration snippets, not just containerlab-specific files. This change does not affect any _netlab_ functionality or files specified in **clab.config_template** that _netlab_ maps into containers with **clab.binds** list (which is adjusted accordingly).

* Internal node attribute **clab._template_cache** has been renamed to **_template_cache** to reflect its provider-independent nature.

### Default Values and Behavior Changes

* Default Cisco IOL (IOS on Linux) container image has been updated to release 17.16.01a, which includes SRv6 support. If you need to use the previous version, you can specify it explicitly in your lab topology using the `clab.image` node attribute.

* For devices with SSH containers (as opposed to VMs), _netlab_ now uses **mgmt.ipv4** as the **ansible_host** value instead of relying on `/etc/hosts` entries. This ensures consistent SSH connectivity across containers and VMs.

### Ansible Compatibility

* _netlab_ now includes workarounds for changes in Ansible 12 and 13:
  * The `set_fact` module now takes precedence over `vars` in Ansible 13, affecting how variables are defined in playbooks.
  * Device readiness checks have been updated to use `is truthy` tests to satisfy Ansible 13 requirements.
  * The **netlab config** command has been completely rewritten to work with Ansible 12+ while maintaining backward compatibility.
